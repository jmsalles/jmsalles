# Guia completo: inventario-homelab-api com FastAPI, PostgreSQL, Vault, Nexus, Gitea Actions, DNS e Nginx Reverse Proxy

## 1. Objetivo

Este documento descreve a criação da aplicação:

```text
inventario-homelab-api
```

A aplicação será uma API REST para inventariar recursos do homelab JMSalles, como:

* Hosts
* Mini PCs
* Servidores
* Serviços
* URLs
* Portas
* Funções de cada equipamento
* Status lógico dos recursos

O fluxo final será:

```text
Gitea
|
Gitea Actions
|
Vault
|
Nexus Registry
|
Docker HML
|
FastAPI
|
PostgreSQL
|
Nginx Reverse Proxy
|
HTTPS
```

Nesta aplicação, não será usado Portainer.

URL final:

```text
https://inventario-hml.jmsalles.homelab.br
```

Documentação Swagger:


## 4. Como o Vault será usado

O Vault será usado pela pipeline para armazenar os segredos da aplicação.

O Gitea Actions fará login no Vault usando AppRole e buscará os dados necessários para:

* Fazer login no Nexus Registry
* Gerar o `.env` da aplicação
* Definir usuário do PostgreSQL
* Definir senha do PostgreSQL
* Definir nome do banco

O Gitea guardará apenas:

```text
VAULT_ADDR
VAULT_ROLE_ID
VAULT_SECRET_ID
```

O Vault guardará:

```text
registry_url
registry_user
registry_password
postgres_db
postgres_user
postgres_password
app_env
```

Caminho no Vault:

```text
secret/homelab/gitea/inventario-homelab-api
```

Como o KV é v2, a API HTTP acessará:

```text
secret/data/homelab/gitea/inventario-homelab-api
```

Importante:

Dentro do container do Vault, usar:

```bash
export VAULT_ADDR='http://127.0.0.1:8200'
```

E usar:

```bash
vi
```

No Gitea Actions, o secret `VAULT_ADDR` deve apontar para um endereço acessível pelo runner. No seu ambiente, pode ser:

```text
https://vault.jmsalles.homelab.br
```

ou outro endereço que o container do job consiga acessar.

```python
from sqlalchemy import create_engine
from sqlalchemy.engine import URL
from sqlalchemy.orm import declarative_base, sessionmaker


DB_HOST = os.getenv("DB_HOST", "inventario-homelab-db")
DB_PORT = int(os.getenv("DB_PORT", "5432"))
DB_NAME = os.getenv("DB_NAME", "inventario_homelab")
DB_USER = os.getenv("DB_USER", "inventario")
DB_PASSWORD = os.getenv("DB_PASSWORD", "inventario")
DATABASE_URL = os.getenv("DATABASE_URL")


if DATABASE_URL:
    database_url = DATABASE_URL
else:
    database_url = URL.create(
        drivername="postgresql+psycopg2",
        username=DB_USER,
        password=DB_PASSWORD,
        host=DB_HOST,
        port=DB_PORT,
        database=DB_NAME,
    )


engine = create_engine(
    database_url,
    pool_pre_ping=True,
)

SessionLocal = sessionmaker(
    autocommit=False,
    autoflush=False,
    bind=engine,
)

Base = declarative_base()


def get_db():
    db = SessionLocal()

    try:
        yield db
    finally:
        db.close()
```

## 17. Criar router de hosts

```bash
vi /home/jmsalles/inventario-homelab-api/app/routers/hosts.py
```

Conteúdo:

```python
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy.orm import Session

from app import models, schemas
from app.database import get_db


router = APIRouter()


@router.get("/", response_model=list[schemas.HostOut])
def list_hosts(db: Session = Depends(get_db)):
    return db.query(models.Host).order_by(models.Host.name).all()


@router.get("/{host_id}", response_model=schemas.HostOut)
def get_host(host_id: int, db: Session = Depends(get_db)):
    host = db.query(models.Host).filter(models.Host.id == host_id).first()

    if not host:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail="Host não encontrado",
        )

    return host


@router.post("/", response_model=schemas.HostOut, status_code=status.HTTP_201_CREATED)
def create_host(payload: schemas.HostCreate, db: Session = Depends(get_db)):
    existing = db.query(models.Host).filter(models.Host.name == payload.name).first()

    if existing:
        raise HTTPException(
            status_code=status.HTTP_409_CONFLICT,
            detail="Já existe um host com esse nome",
        )

    host = models.Host(**payload.model_dump())
    db.add(host)
    db.commit()
    db.refresh(host)

    return host
```

Conteúdo:

```python
import time
from contextlib import asynccontextmanager

from fastapi import FastAPI
from sqlalchemy.exc import OperationalError

from app.database import Base, engine
from app.routers import hosts, services


def init_db_with_retry():
    max_attempts = 30

    for attempt in range(1, max_attempts + 1):
        try:
            Base.metadata.create_all(bind=engine)
            return
        except OperationalError:
            print(f"Banco ainda indisponível. Tentativa {attempt}/{max_attempts}")
            time.sleep(2)

    raise RuntimeError("Não foi possível conectar ao banco após várias tentativas")


@asynccontextmanager
async def lifespan(app: FastAPI):
    init_db_with_retry()
    yield


app = FastAPI(
    title="Inventário Homelab API",
    description="API para inventário de hosts e serviços do homelab JMSalles",
    version="1.0.0",
    lifespan=lifespan,
)


@app.get("/")
def root():
    return {
        "service": "inventario-homelab-api",
        "status": "online",
        "docs": "/docs",
        "health": "/health",
        "hosts": "/api/hosts/",
        "services": "/api/services/",
    }


@app.get("/health")
def health():
    return {
        "status": "ok",
        "service": "inventario-homelab-api",
    }


app.include_router(
    hosts.router,
    prefix="/api/hosts",
    tags=["hosts"],
)

app.include_router(
    services.router,
    prefix="/api/services",
    tags=["services"],
)
```

## 34. Criar workflow Gitea Actions corrigido

Este workflow foi ajustado para evitar erro de YAML como:

```text
yaml: line 160: could not find expected ':'
```

A correção principal é não usar heredoc para gerar arquivos dentro do YAML. Em vez disso, usamos `printf`.

Criar o arquivo:

```bash
vi /home/jmsalles/inventario-homelab-api/.gitea/workflows/build-push-deploy.yml
```

Conteúdo completo:

```yaml
name: Build Push Deploy Inventario Homelab API

on:
  push:
    branches:
      - main

jobs:
  build-push-deploy:
    runs-on: docker-hml

    defaults:
      run:
        shell: sh

    env:
      GIT_SSL_CAINFO: /etc/ssl/certs/ca-certificates.crt
      NODE_EXTRA_CA_CERTS: /etc/ssl/certs/ca-certificates.crt
      VAULT_SECRET_PATH: secret/data/homelab/gitea/inventario-homelab-api
      DEPLOY_DIR: /deploy/inventario-homelab-api
      HOST_DEPLOY_DIR: /home/jmsalles/docker-apps/inventario-homelab-api
      API_CONTAINER_NAME: inventario-homelab-api
      DB_CONTAINER_NAME: inventario-homelab-db
      APP_PORT: 18082

    steps:
      - name: Preparar ambiente do job
        run: |
          apk add --no-cache git nodejs npm bash ca-certificates docker-cli docker-cli-compose curl jq
          cp /certs/jmsalles-homelab-rootCA.crt /usr/local/share/ca-certificates/jmsalles-homelab-rootCA.crt
          update-ca-certificates
          git config --global http.sslCAInfo /etc/ssl/certs/ca-certificates.crt
          git config --global http.sslVerify true
          git --version
          node --version
          docker version
          docker compose version
          curl --version
          jq --version

      - name: Checkout do codigo
        uses: actions/checkout@v4

      - name: Validar workspace
        run: |
          echo "Diretorio atual:"
          pwd

          echo "Arquivos encontrados:"
          find . -maxdepth 5 -type f | sort

          test -f Dockerfile || {
            echo "ERRO: Dockerfile nao encontrado"
            exit 1
          }

          test -f requirements.txt || {
            echo "ERRO: requirements.txt nao encontrado"
            exit 1
          }

          test -f app/main.py || {
            echo "ERRO: app/main.py nao encontrado"
            exit 1
          }

          test -f deploy/docker-compose.template.yml || {
            echo "ERRO: deploy/docker-compose.template.yml nao encontrado"
            exit 1
          }

      - name: Autenticar no Vault e carregar segredos
        run: |
          echo "Autenticando no Vault via AppRole..."

          LOGIN_PAYLOAD=$(jq -n --arg role_id "${{ secrets.VAULT_ROLE_ID }}" --arg secret_id "${{ secrets.VAULT_SECRET_ID }}" '{role_id: $role_id, secret_id: $secret_id}')

          VAULT_LOGIN_RESPONSE=$(curl -fsS --request POST --header "Content-Type: application/json" --data "${LOGIN_PAYLOAD}" "${{ secrets.VAULT_ADDR }}/v1/auth/approle/login")

          VAULT_TOKEN=$(echo "${VAULT_LOGIN_RESPONSE}" | jq -r '.auth.client_token')

          if [ -z "${VAULT_TOKEN}" ] || [ "${VAULT_TOKEN}" = "null" ]; then
            echo "ERRO: token do Vault nao foi obtido"
            exit 1
          fi

          echo "Lendo segredos do Vault..."

          VAULT_SECRET_RESPONSE=$(curl -fsS --header "X-Vault-Token: ${VAULT_TOKEN}" "${{ secrets.VAULT_ADDR }}/v1/${VAULT_SECRET_PATH}")

          REGISTRY_URL=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.registry_url')
          REGISTRY_USER=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.registry_user')
          REGISTRY_PASSWORD=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.registry_password')
          POSTGRES_DB=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.postgres_db')
          POSTGRES_USER=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.postgres_user')
          POSTGRES_PASSWORD=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.postgres_password')
          APP_ENV=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.app_env')

          for item in REGISTRY_URL REGISTRY_USER REGISTRY_PASSWORD POSTGRES_DB POSTGRES_USER POSTGRES_PASSWORD APP_ENV; do
            value=$(eval "printf '%s' \"\${${item}}\"")

            if [ -z "${value}" ] || [ "${value}" = "null" ]; then
              echo "ERRO: ${item} ausente no Vault"
              exit 1
            fi
          done

          echo "REGISTRY_URL=${REGISTRY_URL}" >> "${GITEA_ENV}"
          echo "REGISTRY_USER=${REGISTRY_USER}" >> "${GITEA_ENV}"
          echo "REGISTRY_PASSWORD=${REGISTRY_PASSWORD}" >> "${GITEA_ENV}"
          echo "POSTGRES_DB=${POSTGRES_DB}" >> "${GITEA_ENV}"
          echo "POSTGRES_USER=${POSTGRES_USER}" >> "${GITEA_ENV}"
          echo "POSTGRES_PASSWORD=${POSTGRES_PASSWORD}" >> "${GITEA_ENV}"
          echo "APP_ENV=${APP_ENV}" >> "${GITEA_ENV}"

          echo "Segredos carregados com sucesso."

      - name: Definir variaveis da imagem
        run: |
          echo "IMAGE_NAME=${REGISTRY_URL}/inventario-homelab-api" >> "${GITEA_ENV}"
          echo "IMAGE_TAG=${{ gitea.sha }}" >> "${GITEA_ENV}"

      - name: Login no Nexus Registry
        run: |
          echo "${REGISTRY_PASSWORD}" | docker login "${REGISTRY_URL}" -u "${REGISTRY_USER}" --password-stdin

      - name: Build da imagem
        run: |
          docker build -t "${IMAGE_NAME}:${IMAGE_TAG}" -t "${IMAGE_NAME}:latest" .

      - name: Push da imagem
        run: |
          docker push "${IMAGE_NAME}:${IMAGE_TAG}"
          docker push "${IMAGE_NAME}:latest"

      - name: Criar estrutura de deploy
        run: |
          mkdir -p "${DEPLOY_DIR}/postgres" "${DEPLOY_DIR}/logs" "${DEPLOY_DIR}/config"
          ls -lah "${DEPLOY_DIR}"

      - name: Gerar arquivo env do deploy
        run: |
          umask 077

          printf '%s\n' "APP_ENV=${APP_ENV}" > "${DEPLOY_DIR}/.env"
          printf '%s\n' "" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "POSTGRES_DB=${POSTGRES_DB}" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "POSTGRES_USER=${POSTGRES_USER}" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "POSTGRES_PASSWORD=${POSTGRES_PASSWORD}" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "DB_HOST=${DB_CONTAINER_NAME}" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "DB_PORT=5432" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "DB_NAME=${POSTGRES_DB}" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "DB_USER=${POSTGRES_USER}" >> "${DEPLOY_DIR}/.env"
          printf '%s\n' "DB_PASSWORD=${POSTGRES_PASSWORD}" >> "${DEPLOY_DIR}/.env"

          chmod 600 "${DEPLOY_DIR}/.env"

          echo "Arquivo env gerado em ${DEPLOY_DIR}/.env"
          ls -l "${DEPLOY_DIR}/.env"

      - name: Gerar docker compose do deploy
        run: |
          printf '%s\n' "services:" > "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "  inventario-homelab-db:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    image: postgres:16-alpine" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    container_name: ${DB_CONTAINER_NAME}" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    restart: unless-stopped" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    env_file:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      - .env" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    volumes:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      - ${HOST_DEPLOY_DIR}/postgres:/var/lib/postgresql/data" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    healthcheck:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      test: [\"CMD-SHELL\", \"pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}\"]" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      interval: 10s" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      timeout: 5s" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      retries: 5" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "  inventario-homelab-api:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    image: ${IMAGE_NAME}:${IMAGE_TAG}" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    container_name: ${API_CONTAINER_NAME}" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    restart: unless-stopped" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    env_file:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      - .env" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    ports:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      - \"${APP_PORT}:8000\"" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "    depends_on:" >> "${DEPLOY_DIR}/docker-compose.yml"
          printf '%s\n' "      - inventario-homelab-db" >> "${DEPLOY_DIR}/docker-compose.yml"

          cat "${DEPLOY_DIR}/docker-compose.yml"

      - name: Deploy no Docker HML
        run: |
          docker compose --project-name inventario-homelab-api --project-directory "${DEPLOY_DIR}" -f "${DEPLOY_DIR}/docker-compose.yml" pull

          docker compose --project-name inventario-homelab-api --project-directory "${DEPLOY_DIR}" -f "${DEPLOY_DIR}/docker-compose.yml" up -d

      - name: Validar deploy
        run: |
          docker ps --filter "name=${API_CONTAINER_NAME}"
          docker ps --filter "name=${DB_CONTAINER_NAME}"

          docker compose --project-name inventario-homelab-api --project-directory "${DEPLOY_DIR}" -f "${DEPLOY_DIR}/docker-compose.yml" ps

          echo "Aguardando API responder..."

          for i in $(seq 1 30); do
            if docker exec "${API_CONTAINER_NAME}" python -c "import urllib.request; print(urllib.request.urlopen('http://127.0.0.1:8000/health', timeout=5).read().decode())" | grep -q "ok"; then
              echo "API validada com sucesso."
              exit 0
            fi

            echo "Tentativa ${i}/30..."
            sleep 2
          done

          echo "ERRO: API nao respondeu no prazo esperado"
          docker logs "${API_CONTAINER_NAME}" --tail=100 || true
          docker logs "${DB_CONTAINER_NAME}" --tail=100 || true
          exit 1

      - name: Logout do registry
        run: |
          docker logout "${REGISTRY_URL}"
```

```nginx
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```
