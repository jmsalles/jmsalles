# Guia de criação e deploy do Portal Homelab Nginx com Gitea Actions

## 1. Objetivo

Este documento descreve a criação de uma nova aplicação estática chamada `portal-homelab-nginx`, hospedada em um novo repositório no Gitea e publicada automaticamente no servidor Docker HML usando Gitea Actions.

A aplicação será servida por um container Nginx padrão, sem build de imagem customizada.

O objetivo é criar um portal estático com:

* Múltiplas páginas HTML
* CSS separado
* JavaScript separado
* JSON de dados
* Página de topologia do homelab
* Imagem SVG da topologia
* Tabela de recursos de hardware
* Tabela de recursos de software e serviços
* Funcionalidades simples com JavaScript
* Deploy automático via pipeline
* Mapeamento dos arquivos do site por volume no container Nginx

Nesta etapa não será criada nova entrada DNS e não será criada entrada no Nginx reverse proxy.

O acesso será feito diretamente pelo Docker HML:

```text
http://docker-hml.jmsalles.homelab.br:18081
```

Também pode ser acessado por IP:

```text
http://192.168.31.37:18081
```


## 4. Observação sobre coexistência com a aplicação app-hello-homelab

Este portal foi configurado para conviver com a aplicação anterior `app-hello-homelab`.

Aplicação anterior:

```text
app-hello-homelab
http://docker-hml.jmsalles.homelab.br:18080
```

Nova aplicação:

```text
portal-homelab-nginx
http://docker-hml.jmsalles.homelab.br:18081
```

As duas usam subdiretórios diferentes no host:

```text
/home/jmsalles/docker-apps/app-hello-homelab
/home/jmsalles/docker-apps/portal-homelab-nginx
```

Não apagar o diretório base abaixo, pois ele é compartilhado pelas pipelines:

```text
/home/jmsalles/docker-apps
```

## 5. Pré-requisitos

Antes de iniciar, o ambiente precisa ter:

* Gitea funcionando
* Gitea Actions habilitado
* Gitea Runner registrado
* Runner com label `docker-hml`
* Docker funcionando no Docker HML
* Docker Compose funcionando
* CA interna configurada no runner
* Diretório `/home/jmsalles/docker-apps` montado para os jobs do runner
* Git instalado no Docker HML
* URL `docker-hml.jmsalles.homelab.br` já resolvendo para o Docker HML

Validar Docker:

```bash
docker --version
```

```bash
docker compose version
```

```bash
docker ps
```

Validar Git:

```bash
git --version
```

Validar resolução do Docker HML:

```bash
nslookup docker-hml.jmsalles.homelab.br
```

Validar acesso ao host:

```bash
ping docker-hml.jmsalles.homelab.br
```

## 7. Configurar Git no Docker HML

No servidor Docker HML, configure nome e e-mail do Git.

```bash
git config --global user.name "Jeferson Salles"
```

```bash
git config --global user.email "jefersonmattossalles@gmail.com"
```

Como o Gitea usa certificado da CA interna do homelab, configure o Git para confiar nessa CA.

```bash
git config --global http.sslCAInfo /home/jmsalles/nginx/ca/jmsalles-homelab-rootCA.crt
```

Validar:

```bash
git config --global --list
```

```javascript
async function carregarServicos() {
  const lista = document.getElementById("listaServicos");
  const filtro = document.getElementById("filtroServicos");

  if (!lista) {
    return;
  }

  const resposta = await fetch("/data/servicos.json");
  const servicos = await resposta.json();

  function renderizar(valorFiltro) {
    lista.innerHTML = "";

    const termo = (valorFiltro || "").toLowerCase();

    const filtrados = servicos.filter((servico) => {
      return (
        servico.nome.toLowerCase().includes(termo) ||
        servico.descricao.toLowerCase().includes(termo) ||
        servico.categoria.toLowerCase().includes(termo)
      );
    });

    filtrados.forEach((servico) => {
      const card = document.createElement("article");
      card.innerHTML = `
        <h2>${servico.nome}</h2>
        <p>${servico.descricao}</p>
        <p><strong>Categoria:</strong> ${servico.categoria}</p>
      `;
      lista.appendChild(card);
    });

    if (filtrados.length === 0) {
      lista.innerHTML = "<p>Nenhum serviço encontrado.</p>";
    }
  }

  renderizar("");

  if (filtro) {
    filtro.addEventListener("input", () => {
      renderizar(filtro.value);
    });
  }
}
```

## 22. Ajuste necessário no Gitea Runner

Para que a pipeline consiga copiar os arquivos para o host Docker HML, o diretório `/home/jmsalles/docker-apps` precisa estar montado nos containers de job.

Esse ajuste também mantém a compatibilidade com a aplicação antiga `app-hello-homelab`, pois cada aplicação usa seu próprio subdiretório.

No Docker HML, crie a estrutura base.

```bash
mkdir -p /home/jmsalles/docker-apps/{portal-homelab-nginx}
```

Ajuste permissões.

```bash
chown -R jmsalles:jmsalles /home/jmsalles/docker-apps
```

Faça backup do arquivo do runner antes de alterar.

```bash
cd /home/jmsalles/gitea-runner
```

```bash
cp -av config.yaml config.yaml.bkp-$(date +%F-%H%M%S)
```

Edite o `config.yaml` do Gitea Runner.

```bash
vim /home/jmsalles/gitea-runner/config.yaml
```

No bloco `container`, deixe assim:

```yaml
container:
  network: ""
  privileged: false
  options: "-v /home/jmsalles/gitea-runner/certs:/certs:ro -v /home/jmsalles/docker-apps:/deploy:rw -e GIT_SSL_CAINFO=/certs/jmsalles-homelab-rootCA.crt -e NODE_EXTRA_CA_CERTS=/certs/jmsalles-homelab-rootCA.crt"
  workdir_parent:
  valid_volumes:
    - /home/jmsalles/gitea-runner/certs
    - /home/jmsalles/docker-apps
```

Não adicione manualmente o Docker socket nesse bloco.

Não use:


O Docker socket deve estar apenas no `docker-compose.yml` do runner.


```bash

```

Valide logs.

```bash
docker compose logs --tail=80
```

## 23. Criar workflow de deploy sem build

Este workflow não faz `docker build` e não faz push para registry.

Ele apenas:

* Faz checkout do código
* Copia os arquivos estáticos para o host
* Gera o `docker-compose.yml`
* Sobe o container `nginx:alpine`
* Valida o acesso HTTP dentro do próprio container Nginx

Crie o workflow:

```bash
vim /home/jmsalles/portal-homelab-nginx/.gitea/workflows/deploy.yml
```

Conteúdo:

```yaml
name: Deploy Portal Homelab Nginx

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: docker-hml

    defaults:
      run:
        shell: sh

    env:
      GIT_SSL_CAINFO: /etc/ssl/certs/ca-certificates.crt
      NODE_EXTRA_CA_CERTS: /etc/ssl/certs/ca-certificates.crt

      DEPLOY_DIR: /deploy/portal-homelab-nginx
      SITE_DIR: /deploy/portal-homelab-nginx/site

      HOST_DEPLOY_DIR: /home/jmsalles/docker-apps/portal-homelab-nginx
      HOST_SITE_DIR: /home/jmsalles/docker-apps/portal-homelab-nginx/site

      CONTAINER_NAME: portal-homelab-nginx
      APP_PORT: 18081

    steps:
      - name: Preparar ambiente do job
        run: |
          apk add --no-cache git nodejs npm bash ca-certificates docker-cli-compose curl rsync
          cp /certs/jmsalles-homelab-rootCA.crt /usr/local/share/ca-certificates/jmsalles-homelab-rootCA.crt
          update-ca-certificates
          git config --global http.sslCAInfo /etc/ssl/certs/ca-certificates.crt
          git config --global http.sslVerify true
          git --version
          node --version
          docker version
          docker compose version

      - name: Checkout do codigo
        uses: actions/checkout@v4

      - name: Validar estrutura do site
        run: |
          echo "Diretorio atual:"
          pwd

          echo "Arquivos do repositorio:"
          find . -maxdepth 4 -type f | sort

          test -f site/index.html || {
            echo "ERRO: site/index.html nao encontrado"
            exit 1
          }

          test -f site/pages/topologia.html || {
            echo "ERRO: pagina de topologia nao encontrada"
            exit 1
          }

          test -f site/assets/img/topologia-homelab.svg || {
            echo "ERRO: imagem SVG da topologia nao encontrada"
            exit 1
          }

          test -f site/assets/css/style.css || {
            echo "ERRO: CSS principal nao encontrado"
            exit 1
          }

          test -f site/assets/js/app.js || {
            echo "ERRO: JavaScript principal nao encontrado"
            exit 1
          }

      - name: Criar estrutura de deploy
        run: |
          mkdir -p "${DEPLOY_DIR}"/{site,logs,config}
          mkdir -p "${SITE_DIR}"/{pages,assets/{css,js,img},data}

          echo "Teste de escrita no /deploy:"
          echo "teste-mount-deploy" > /deploy/teste-mount.txt
          ls -lah /deploy

          echo "Estrutura do deploy:"
          ls -lah "${DEPLOY_DIR}"

      - name: Publicar arquivos estaticos no host
        run: |
          rsync -av --delete site/ "${SITE_DIR}/"

          echo "Arquivos publicados em ${SITE_DIR}:"
          find "${SITE_DIR}" -maxdepth 4 -type f | sort

          echo "Validando index.html no destino:"
          test -f "${SITE_DIR}/index.html" || {
            echo "ERRO: index.html nao foi publicado em ${SITE_DIR}"
            ls -lah "${SITE_DIR}"
            exit 1
          }

      - name: Gerar docker-compose.yml
        run: |
          cat > "${DEPLOY_DIR}/docker-compose.yml" <<EOF
          services:
            portal-homelab-nginx:
              image: nginx:alpine
              container_name: ${CONTAINER_NAME}
              restart: unless-stopped
              ports:
                - "${APP_PORT}:80"
              volumes:
                - ${HOST_SITE_DIR}:/usr/share/nginx/html:ro
          EOF

          cat "${DEPLOY_DIR}/docker-compose.yml"

      - name: Deploy no Docker HML
        run: |
          docker compose \
            --project-name portal-homelab-nginx \
            -f "${DEPLOY_DIR}/docker-compose.yml" \
            pull

          docker compose \
            --project-name portal-homelab-nginx \
            -f "${DEPLOY_DIR}/docker-compose.yml" \
            up -d --force-recreate

      - name: Validar deploy
        run: |
          docker ps --filter "name=${CONTAINER_NAME}"

          docker compose \
            --project-name portal-homelab-nginx \
            -f "${DEPLOY_DIR}/docker-compose.yml" \
            ps

          sleep 3

          echo "Validando arquivos dentro do container:"
          docker exec "${CONTAINER_NAME}" ls -lah /usr/share/nginx/html

          echo "Validando index.html dentro do container:"
          docker exec "${CONTAINER_NAME}" test -f /usr/share/nginx/html/index.html

          echo "Validando pagina inicial:"
          docker exec "${CONTAINER_NAME}" wget -qO- http://127.0.0.1 | grep "Portal Homelab JMSalles"

          echo "Validando pagina de topologia:"
          docker exec "${CONTAINER_NAME}" wget -qO- http://127.0.0.1/pages/topologia.html | grep "Topologia do Homelab JMSalles"
```

## 24. Por que o volume precisa usar HOST_SITE_DIR

Dentro do job, o diretório abaixo existe porque foi montado pelo runner:

```text
/deploy/portal-homelab-nginx/site
```

Mas o container Nginx é criado pelo Docker daemon do host.

Por isso, no `docker-compose.yml` gerado, o volume precisa apontar para o caminho real do host:

```text
/home/jmsalles/docker-apps/portal-homelab-nginx/site
```

Correto:

```yaml
volumes:
  - /home/jmsalles/docker-apps/portal-homelab-nginx/site:/usr/share/nginx/html:ro
```

Incorreto:

```yaml
volumes:
  - /deploy/portal-homelab-nginx/site:/usr/share/nginx/html:ro
```

Se usar `/deploy` no volume do Nginx, o container sobe, mas o diretório fica vazio e o Nginx retorna:

```text
403 Forbidden
```

## 36. Troubleshooting

### 36.1 Pipeline falha no checkout por certificado

Erro possível:

```text
certificate signed by unknown authority
```

Validar se a CA está montada no job pelo `config.yaml` do runner:

```yaml
container:
  options: "-v /home/jmsalles/gitea-runner/certs:/certs:ro -v /home/jmsalles/docker-apps:/deploy:rw -e GIT_SSL_CAINFO=/certs/jmsalles-homelab-rootCA.crt -e NODE_EXTRA_CA_CERTS=/certs/jmsalles-homelab-rootCA.crt"
```

Validar se o workflow instala a CA:

```bash
cp /certs/jmsalles-homelab-rootCA.crt /usr/local/share/ca-certificates/jmsalles-homelab-rootCA.crt
update-ca-certificates
```

---

### 36.2 Erro bash not found

Erro:

```text
exec: "bash": executable file not found in $PATH
```

Causa:

A label `docker-hml` usa imagem baseada em Alpine.

Solução no workflow:

```yaml
defaults:
  run:
    shell: sh
```

---

### 36.3 Erro de volume não encontrado

Erro possível:

```text
invalid mount config
```

Validar se o diretório existe no host:

```bash
ls -lah /home/jmsalles/docker-apps
```

Criar novamente:

```bash
mkdir -p /home/jmsalles/docker-apps/{portal-homelab-nginx}
```

Ajustar dono:

```bash
chown -R jmsalles:jmsalles /home/jmsalles/docker-apps
```

---

### 36.4 Erro de Docker socket duplicado

Erro:

```text
Duplicate mount point: /var/run/docker.sock
```

Causa:

O socket Docker foi adicionado manualmente no `container.options`.

Solução:

Não colocar isto no `container.options`:

```text
-v /var/run/docker.sock:/var/run/docker.sock
```

O socket deve estar no `docker-compose.yml` do runner:

```yaml
volumes:
  - /var/run/docker.sock:/var/run/docker.sock
```

---

### 36.5 Container sobe, mas retorna 403 Forbidden

Erro:

```text
HTTP/1.1 403 Forbidden
```

Causa provável:

O Nginx subiu, mas o diretório `/usr/share/nginx/html` está vazio ou sem `index.html`.

Validar:

```bash
docker exec portal-homelab-nginx ls -lah /usr/share/nginx/html
```

Se aparecer vazio, valide o diretório do host:

```bash
ls -lah /home/jmsalles/docker-apps/portal-homelab-nginx/site
```

Precisa aparecer:

```text
index.html
assets
data
pages
```

Também valide se o volume no `docker-compose.yml` usa o caminho real do host:

```bash
cat /home/jmsalles/docker-apps/portal-homelab-nginx/docker-compose.yml
```

Correto:

```yaml
volumes:
  - /home/jmsalles/docker-apps/portal-homelab-nginx/site:/usr/share/nginx/html:ro
```

Incorreto:

```yaml
volumes:
  - /deploy/portal-homelab-nginx/site:/usr/share/nginx/html:ro
```
