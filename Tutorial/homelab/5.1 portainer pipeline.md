# Guia corrigido: app-hello-portainer com Gitea Actions, Vault, Nexus e Portainer API

## 1. Objetivo

Este documento descreve a criação do projeto:

```text
app-hello-portainer
```

O projeto será novo e independente dos projetos existentes.

Projetos existentes que não serão alterados:

```text
app-hello-homelab
portal-homelab-nginx
```

Novo projeto:

```text
app-hello-portainer
```

O objetivo é validar o fluxo:

```text
Gitea
|
Gitea Actions
|
Vault
|
Nexus Registry
|
Portainer API
|
Docker HML
|
app-hello-portainer
```

Nesta abordagem, não será usado webhook do Portainer, pois o webhook de stack não está disponível no Portainer CE.

A pipeline fará:

```text
1. Checkout do código
2. Login no Vault via AppRole
3. Leitura dos segredos no Vault
4. Build da imagem Docker
5. Push da imagem para o Nexus
6. Atualização da stack via Portainer API
7. Validação do container
```

A aplicação ficará disponível em:

```text
http://docker-hml.jmsalles.homelab.br:18082
```

## 15. Configurar Vault para o projeto

### 15.1 Autenticar no Vault

No host onde você administra o Vault:

```bash
export VAULT_ADDR="https://vault.jmsalles.homelab.br"
```

Valide:

```bash
vault status
```

Login administrativo:

```bash
vault login
```

---

### 15.2 Validar se o KV está ativo

```bash
vault secrets list
```

Procure por:

```text
secret/
```

Se não existir e você quiser criar em KV v2:

```bash
vault secrets enable -path=secret kv-v2
```

---

### 15.3 Criar policy do projeto

Crie o arquivo:

```bash
vim /tmp/gitea-app-hello-portainer-policy.hcl
```

Conteúdo:

```hcl
path "secret/data/homelab/gitea/app-hello-portainer" {
  capabilities = ["read"]
}

path "secret/metadata/homelab/gitea/app-hello-portainer" {
  capabilities = ["read"]
}
```

Aplicar policy:

```bash
vault policy write gitea-app-hello-portainer /tmp/gitea-app-hello-portainer-policy.hcl
```

Validar:

```bash
vault policy read gitea-app-hello-portainer
```

---

### 15.4 Habilitar AppRole se ainda não estiver habilitado

Validar auth methods:

```bash
vault auth list
```

Se `approle/` não existir:

```bash
vault auth enable approle
```

---

### 15.5 Criar AppRole para a pipeline

```bash
vault write auth/approle/role/gitea-app-hello-portainer \
  token_policies="gitea-app-hello-portainer" \
  token_ttl="1h" \
  token_max_ttl="4h" \
  secret_id_ttl="0"
```

Observação:

Para homelab, `secret_id_ttl="0"` simplifica porque o SecretID não expira. Em ambiente produtivo, o ideal é usar TTL e rotação periódica.

---

### 15.6 Obter RoleID

```bash
vault read -field=role_id auth/approle/role/gitea-app-hello-portainer/role-id
```

Guarde o valor para criar o secret no Gitea:

```text
VAULT_ROLE_ID
```

---

### 15.7 Gerar SecretID

```bash
vault write -f -field=secret_id auth/approle/role/gitea-app-hello-portainer/secret-id
```

Guarde o valor para criar o secret no Gitea:

```text
VAULT_SECRET_ID
```

## 20. Descobrir Endpoint ID e Stack ID do Portainer

No Docker HML, instale `jq`, se necessário:

```bash
sudo dnf install -y jq
```

Defina as variáveis temporárias:

```bash
export PORTAINER_URL="https://portainer-hml.jmsalles.homelab.br"
```

```bash
export PORTAINER_API_TOKEN="COLE_AQUI_O_TOKEN_DO_PORTAINER"
```

Listar stacks:

```bash
curl -ks \
  -H "X-API-Key: ${PORTAINER_API_TOKEN}" \
  "${PORTAINER_URL}/api/stacks" | jq
```

Filtrar a stack `app-hello-portainer`:

```bash
curl -ks \
  -H "X-API-Key: ${PORTAINER_API_TOKEN}" \
  "${PORTAINER_URL}/api/stacks" \
  | jq '.[] | select(.Name=="app-hello-portainer") | {Id, Name, EndpointId, Type}'
```

Resultado esperado parecido com:

```json
{
  "Id": 2,
  "Name": "app-hello-portainer",
  "EndpointId": 1,
  "Type": 2
}
```

Guarde:

```text
portainer_stack_id
portainer_endpoint_id
```

Exemplo:

```text
portainer_stack_id=2
portainer_endpoint_id=1
```

## 23. Criar workflow Gitea Actions

Crie:

```bash
vim /home/jmsalles/app-hello-portainer/.gitea/workflows/build-push-portainer.yml
```

Conteúdo:

```yaml
name: Build, Push e Deploy via Portainer API com Vault

on:
  push:
    branches:
      - main

jobs:
  build-push-deploy:
    runs-on: docker-hml

    defaults:
      run:
        shell: sh

    env:
      GIT_SSL_CAINFO: /etc/ssl/certs/ca-certificates.crt
      NODE_EXTRA_CA_CERTS: /etc/ssl/certs/ca-certificates.crt
      CONTAINER_NAME: app-hello-portainer
      VAULT_SECRET_PATH: secret/data/homelab/gitea/app-hello-portainer

    steps:
      - name: Preparar ambiente do job
        run: |
          apk add --no-cache git nodejs npm bash ca-certificates docker-cli curl jq
          cp /certs/jmsalles-homelab-rootCA.crt /usr/local/share/ca-certificates/jmsalles-homelab-rootCA.crt
          update-ca-certificates
          git config --global http.sslCAInfo /etc/ssl/certs/ca-certificates.crt
          git config --global http.sslVerify true
          git --version
          node --version
          docker version
          curl --version
          jq --version

      - name: Checkout do codigo
        uses: actions/checkout@v4

      - name: Validar workspace
        run: |
          echo "Diretorio atual:"
          pwd

          echo "Arquivos encontrados:"
          find . -maxdepth 4 -type f | sort

          test -f Dockerfile || {
            echo "ERRO: Dockerfile nao encontrado"
            exit 1
          }

          test -f app/index.html || {
            echo "ERRO: app/index.html nao encontrado"
            exit 1
          }

          test -f deploy/docker-compose.portainer.yml || {
            echo "ERRO: deploy/docker-compose.portainer.yml nao encontrado"
            exit 1
          }

      - name: Autenticar no Vault e carregar segredos
        run: |
          echo "Autenticando no Vault via AppRole..."

          LOGIN_PAYLOAD=$(jq -n \
            --arg role_id "${{ secrets.VAULT_ROLE_ID }}" \
            --arg secret_id "${{ secrets.VAULT_SECRET_ID }}" \
            '{role_id: $role_id, secret_id: $secret_id}')

          VAULT_LOGIN_RESPONSE=$(curl -fsS \
            --request POST \
            --header "Content-Type: application/json" \
            --data "${LOGIN_PAYLOAD}" \
            "${{ secrets.VAULT_ADDR }}/v1/auth/approle/login")

          VAULT_TOKEN=$(echo "${VAULT_LOGIN_RESPONSE}" | jq -r '.auth.client_token')

          if [ -z "${VAULT_TOKEN}" ] || [ "${VAULT_TOKEN}" = "null" ]; then
            echo "ERRO: token do Vault nao foi obtido"
            exit 1
          fi

          echo "Lendo segredos do Vault..."

          VAULT_SECRET_RESPONSE=$(curl -fsS \
            --header "X-Vault-Token: ${VAULT_TOKEN}" \
            "${{ secrets.VAULT_ADDR }}/v1/${VAULT_SECRET_PATH}")

          REGISTRY_URL=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.registry_url')
          REGISTRY_USER=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.registry_user')
          REGISTRY_PASSWORD=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.registry_password')
          PORTAINER_URL=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.portainer_url')
          PORTAINER_API_TOKEN=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.portainer_api_token')
          PORTAINER_ENDPOINT_ID=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.portainer_endpoint_id')
          PORTAINER_STACK_ID=$(echo "${VAULT_SECRET_RESPONSE}" | jq -r '.data.data.portainer_stack_id')

          for item in REGISTRY_URL REGISTRY_USER REGISTRY_PASSWORD PORTAINER_URL PORTAINER_API_TOKEN PORTAINER_ENDPOINT_ID PORTAINER_STACK_ID; do
            value=$(eval echo \$$item)
            if [ -z "${value}" ] || [ "${value}" = "null" ]; then
              echo "ERRO: ${item} ausente no Vault"
              exit 1
            fi
          done

          echo "REGISTRY_URL=${REGISTRY_URL}" >> "${GITEA_ENV}"
          echo "REGISTRY_USER=${REGISTRY_USER}" >> "${GITEA_ENV}"
          echo "REGISTRY_PASSWORD=${REGISTRY_PASSWORD}" >> "${GITEA_ENV}"
          echo "PORTAINER_URL=${PORTAINER_URL}" >> "${GITEA_ENV}"
          echo "PORTAINER_API_TOKEN=${PORTAINER_API_TOKEN}" >> "${GITEA_ENV}"
          echo "PORTAINER_ENDPOINT_ID=${PORTAINER_ENDPOINT_ID}" >> "${GITEA_ENV}"
          echo "PORTAINER_STACK_ID=${PORTAINER_STACK_ID}" >> "${GITEA_ENV}"

          echo "Segredos carregados com sucesso."

      - name: Definir variaveis da imagem
        run: |
          echo "IMAGE_NAME=${REGISTRY_URL}/app-hello-portainer" >> "${GITEA_ENV}"
          echo "IMAGE_TAG=${{ gitea.sha }}" >> "${GITEA_ENV}"

      - name: Login no Nexus Registry
        run: |
          echo "${REGISTRY_PASSWORD}" | docker login "${REGISTRY_URL}" -u "${REGISTRY_USER}" --password-stdin

      - name: Build da imagem
        run: |
          docker build \
            -t "${IMAGE_NAME}:${IMAGE_TAG}" \
            -t "${IMAGE_NAME}:latest" \
            .

      - name: Push da imagem
        run: |
          docker push "${IMAGE_NAME}:${IMAGE_TAG}"
          docker push "${IMAGE_NAME}:latest"

      - name: Renderizar compose para Portainer
        run: |
          echo "Gerando compose com tag imutavel da imagem..."

          sed "s|image: .*app-hello-portainer.*|image: ${IMAGE_NAME}:${IMAGE_TAG}|g" \
            deploy/docker-compose.portainer.yml \
            > /tmp/docker-compose.portainer.rendered.yml

          echo "Compose renderizado:"
          cat /tmp/docker-compose.portainer.rendered.yml

      - name: Atualizar stack via Portainer API
        run: |
          echo "Atualizando stack ${CONTAINER_NAME} via Portainer API..."

          STACK_FILE_CONTENT=$(cat /tmp/docker-compose.portainer.rendered.yml)

          UPDATE_PAYLOAD=$(jq -n \
            --arg stackFileContent "${STACK_FILE_CONTENT}" \
            '{
              StackFileContent: $stackFileContent,
              Env: [],
              Prune: true,
              PullImage: true
            }')

          HTTP_CODE=$(curl -fsS \
            -o /tmp/portainer-update-response.json \
            -w "%{http_code}" \
            -X PUT \
            -H "X-API-Key: ${PORTAINER_API_TOKEN}" \
            -H "Content-Type: application/json" \
            --data "${UPDATE_PAYLOAD}" \
            "${PORTAINER_URL}/api/stacks/${PORTAINER_STACK_ID}?endpointId=${PORTAINER_ENDPOINT_ID}" || true)

          echo "HTTP_CODE=${HTTP_CODE}"

          echo "Resposta do Portainer:"
          cat /tmp/portainer-update-response.json || true

          if [ "${HTTP_CODE}" != "200" ] && [ "${HTTP_CODE}" != "204" ]; then
            echo "ERRO: falha ao atualizar stack via Portainer API"
            exit 1
          fi

          echo "Stack atualizada com sucesso."

      - name: Aguardar e validar deploy
        run: |
          echo "Aguardando redeploy da stack ${CONTAINER_NAME}..."

          for i in $(seq 1 30); do
            if docker ps --format '{{.Names}}' | grep -qx "${CONTAINER_NAME}"; then
              if docker exec "${CONTAINER_NAME}" wget -qO- http://127.0.0.1 | grep -q "App Hello Portainer"; then
                echo "Aplicacao validada com sucesso."
                exit 0
              fi
            fi

            echo "Aguardando aplicacao responder... tentativa ${i}/30"
            sleep 2
          done

          echo "ERRO: aplicacao nao respondeu apos redeploy"
          docker ps --filter "name=${CONTAINER_NAME}"
          docker logs "${CONTAINER_NAME}" --tail=80 || true
          exit 1

      - name: Logout do registry
        run: |
          docker logout "${REGISTRY_URL}"
```

## 30. Troubleshooting

### 30.1 Falha ao autenticar no Vault

Erro provável:

```text
ERRO: token do Vault nao foi obtido
```

Validar no Gitea:

```text
Settings
Actions
Secrets
```

Conferir:

```text
VAULT_ADDR
VAULT_ROLE_ID
VAULT_SECRET_ID
```

Validar AppRole manualmente:

```bash
vault write auth/approle/login \
  role_id="VALOR_DO_ROLE_ID" \
  secret_id="VALOR_DO_SECRET_ID"
```

---

### 30.2 Pipeline não consegue ler segredo no Vault

Erro provável:

```text
permission denied
```

Validar policy:

```bash
vault policy read gitea-app-hello-portainer
```

Validar path:

```bash
vault kv get secret/homelab/gitea/app-hello-portainer
```

Se o KV não estiver em `secret/`, ajuste no workflow:

```yaml
VAULT_SECRET_PATH: secret/data/homelab/gitea/app-hello-portainer
```

Exemplo se o mount for `kv/`:

```yaml
VAULT_SECRET_PATH: kv/data/homelab/gitea/app-hello-portainer
```

---

### 30.3 Login no Nexus falha

Erro provável:

```text
unauthorized
```

Validar no Vault:

```bash
vault kv get -field=registry_user secret/homelab/gitea/app-hello-portainer
```

Validar login manual:

```bash
docker login registry.jmsalles.homelab.br
```

Validar permissões do usuário no Nexus.

---

### 30.4 Push para Nexus falha

Validar imagem:

```bash
docker images | grep app-hello-portainer
```

Validar tag:

```bash
docker images | grep registry.jmsalles.homelab.br/app-hello-portainer
```

Validar se o registry responde:

```bash
curl -k -I https://registry.jmsalles.homelab.br
```

---

### 30.5 API do Portainer retorna 401 ou 403

Erro provável:

```text
Access denied
Unauthorized
```

Validar token no Docker HML:

```bash
export PORTAINER_URL="https://portainer-hml.jmsalles.homelab.br"
```

```bash
export PORTAINER_API_TOKEN="TOKEN_DO_PORTAINER"
```

```bash
curl -ks \
  -H "X-API-Key: ${PORTAINER_API_TOKEN}" \
  "${PORTAINER_URL}/api/stacks" | jq
```

Se falhar, gere novo token no Portainer com usuário que tenha permissão administrativa na stack e no ambiente Docker local.

---

### 30.6 API do Portainer retorna 404

Erro provável:

```text
Unable to find the environment associated to the stack
```

Validar `portainer_endpoint_id` e `portainer_stack_id` no Vault:

```bash
vault kv get -field=portainer_endpoint_id secret/homelab/gitea/app-hello-portainer
```

```bash
vault kv get -field=portainer_stack_id secret/homelab/gitea/app-hello-portainer
```

Comparar com a API:

```bash
curl -ks \
  -H "X-API-Key: ${PORTAINER_API_TOKEN}" \
  "${PORTAINER_URL}/api/stacks" \
  | jq '.[] | select(.Name=="app-hello-portainer") | {Id, Name, EndpointId, Type}'
```

Atualize o Vault se os valores estiverem incorretos.

---

### 30.7 API do Portainer retorna 400

Valide o compose renderizado na pipeline.

No job, procure a saída:

```text
Compose renderizado:
```

O conteúdo precisa estar parecido com:

```yaml
services:
  app-hello-portainer:
    image: registry.jmsalles.homelab.br/app-hello-portainer:<sha-do-commit>
    container_name: app-hello-portainer
    restart: unless-stopped
    ports:
      - "18082:80"
```
