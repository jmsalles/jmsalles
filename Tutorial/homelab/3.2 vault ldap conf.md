# Guia completo de autenticação LDAP do Vault com Active Directory

## 1. Objetivo

Este documento descreve como configurar a autenticação LDAP do HashiCorp Vault usando o Active Directory do homelab JMSalles.

O foco é permitir login no Vault usando usuários do AD, como:

```text
jmsalles
adm-jeferson
```

O grupo do AD usado para conceder permissão administrativa no Vault será:

```text
GRP_HOMELAB_ADMINS
```

## 2. Dados do ambiente

| Item                    | Valor                                                                          |
| ----------------------- | ------------------------------------------------------------------------------ |
| Vault URL               | `https://vault.jmsalles.homelab.br`                                            |
| Container Vault         | `vault`                                                                        |
| Host Docker HML         | `docker-hml.jmsalles.homelab.br`                                               |
| Diretório do Vault      | `/home/jmsalles/vault`                                                         |
| Domínio AD              | `jmsalles.homelab.br`                                                          |
| Domain Controller       | `winserver.jmsalles.homelab.br`                                                |

| Porta LDAP              | `389`                                                                          |
| Base DN                 | `DC=jmsalles,DC=homelab,DC=br`                                                 |
| User DN                 | `OU=Homelab,DC=jmsalles,DC=homelab,DC=br`                                      |
| Group DN                | `OU=Groups,OU=Homelab,DC=jmsalles,DC=homelab,DC=br`                            |
| Bind DN                 | `CN=svc-ldap-bind,OU=Service Accounts,OU=Homelab,DC=jmsalles,DC=homelab,DC=br` |
| User attribute          | `sAMAccountName`                                                               |
| Grupo AD                | `GRP_HOMELAB_ADMINS`                                                           |
| Policy Vault            | `homelab-vault-admin`                                                          |

## 3. Visão geral da integração

Fluxo da autenticação:

```text
Usuário acessa o Vault
|
Seleciona método LDAP
|
Informa usuário e senha do AD
|
Vault usa svc-ldap-bind para consultar o AD
|
Vault localiza o usuário pelo sAMAccountName
|
Vault valida a senha no AD
|
Vault consulta os grupos do usuário
|
Vault aplica as policies mapeadas
```

Exemplo de login esperado:

```text
jmsalles
```

ou:

```text
adm-jeferson
```

Não usar inicialmente:

```text
JMSALLES\jmsalles
```

ou:

```text
jmsalles@jmsalles.homelab.br
```

Neste modelo, o login será feito usando o atributo:

```text
sAMAccountName
```

## 4. Observação importante sobre permissões

O Vault não herda automaticamente permissões administrativas do AD.

Mesmo que o usuário seja administrador no AD, ele só terá permissão no Vault se houver uma policy associada ao usuário ou ao grupo LDAP.

Neste guia, o grupo:

```text
GRP_HOMELAB_ADMINS
```

será associado à policy:

```text
homelab-vault-admin
```

## 13. Corrigir ou aplicar a configuração LDAP do Vault

Esta foi a configuração que corrigiu o erro de autenticação no seu ambiente.

O erro anterior ocorria porque a configuração estava ficando com:

```text
userattr = cn
discoverdn = false
userfilter incorreto
```

O correto para seu AD é:

```text
userattr = sAMAccountName
discoverdn = true
userfilter = ({{.UserAttr}}={{.Username}})
```

Dentro do container do Vault, leia a senha do usuário `svc-ldap-bind` em variável:

```bash
read -s LDAP_BINDPASS
```

Digite a senha do usuário `svc-ldap-bind`.

Agora aplique a configuração LDAP:

```bash
vault write auth/ldap/config \
  url="ldap://192.168.31.24:389" \
  binddn="CN=svc-ldap-bind,OU=Service Accounts,OU=Homelab,DC=jmsalles,DC=homelab,DC=br" \
  bindpass="$LDAP_BINDPASS" \
  userdn="OU=Homelab,DC=jmsalles,DC=homelab,DC=br" \
  userattr="sAMAccountName" \
  userfilter="({{.UserAttr}}={{.Username}})" \
  discoverdn=true \
  groupdn="OU=Groups,OU=Homelab,DC=jmsalles,DC=homelab,DC=br" \
  groupattr="cn" \
  groupfilter="(&(objectClass=group)(member={{.UserDN}}))" \
  starttls=false \
  deny_null_bind=true
```

Limpe a variável:

```bash
unset LDAP_BINDPASS
```

## 15. Criar policy administrativa do homelab

Atenção:

```text
Esta policy concede permissão ampla dentro do Vault.
Para homelab, é prática e simples.
Em ambiente corporativo, o ideal seria criar policies menores por aplicação, time ou caminho.
```

Crie o arquivo corretamente usando heredoc.

Importante: o `EOF` precisa ficar sozinho na última linha.

```bash
cat > /tmp/homelab-vault-admin.hcl <<'EOF'
path "*" {
  capabilities = ["create", "read", "update", "delete", "list", "sudo"]
}
EOF
```

Aplicar a policy:

```bash
vault policy write homelab-vault-admin /tmp/homelab-vault-admin.hcl
```

Validar:

```bash
vault policy read homelab-vault-admin
```

Resultado esperado:

```text
path "*" {
  capabilities = ["create", "read", "update", "delete", "list", "sudo"]
}
```

## 16. Criar policy alternativa somente para KV secret

Esta etapa é opcional.

Use esta policy se quiser uma permissão mais controlada apenas para o KV `secret/`.

```bash
cat > /tmp/homelab-kv-admin.hcl <<'EOF'
path "secret/*" {
  capabilities = ["create", "read", "update", "delete", "list"]
}

path "sys/internal/ui/mounts" {
  capabilities = ["read"]
}

path "sys/internal/ui/mounts/*" {
  capabilities = ["read"]
}
EOF
```

Aplicar:

```bash
vault policy write homelab-kv-admin /tmp/homelab-kv-admin.hcl
```

Validar:

```bash
vault policy read homelab-kv-admin
```

Neste guia, seguiremos usando:

```text
homelab-vault-admin
```

## 17. Mapear grupo do AD para policy do Vault

Mapeie o grupo AD:

```text
GRP_HOMELAB_ADMINS
```

para a policy:

```text

```

Execute:

```bash
vault write auth/ldap/groups/GRP_HOMELAB_ADMINS policies=homelab-vault-admin
```

Validar:

```bash
vault read auth/ldap/groups/GRP_HOMELAB_ADMINS
```

Resultado esperado:

```text
policies    [homelab-vault-admin]
```

## 25. Criar policy somente leitura para uso futuro

Se futuramente quiser criar um grupo de leitura, por exemplo:

```text
GRP_VAULT_READONLY
```

crie uma policy assim:

```bash
cat > /tmp/homelab-kv-readonly.hcl <<'EOF'
path "secret/data/*" {
  capabilities = ["read"]
}

path "secret/metadata/*" {
  capabilities = ["list", "read"]
}

path "sys/internal/ui/mounts" {
  capabilities = ["read"]
}

path "sys/internal/ui/mounts/*" {
  capabilities = ["read"]
}
EOF
```

Aplicar:

```bash
vault policy write homelab-kv-readonly /tmp/homelab-kv-readonly.hcl
```

Mapear grupo futuro:

```bash
vault write auth/ldap/groups/GRP_VAULT_READONLY policies=homelab-kv-readonly
```

## 26. Remover mapeamento direto de usuário

Depois que o mapeamento por grupo estiver funcionando, se quiser remover os mapeamentos diretos:




Se não houver usuários mapeados diretamente, pode retornar vazio.



## 28. Troubleshooting

### 28.1 Erro invalid credentials

Erro:

```text
Code: 400
invalid credentials
```

Possíveis causas:

```text
Senha do usuário AD incorreta
Usuário bloqueado no AD
Usuário desabilitado no AD
Usuário exigindo troca de senha
userattr configurado errado
userfilter configurado errado
discoverdn configurado como false indevidamente
Bind DN ou senha do svc-ldap-bind incorretos
```

Validar configuração do Vault:

```bash
vault read auth/ldap/config
```

Valores esperados:

```text
userattr       sAMAccountName
userfilter     ({{.UserAttr}}={{.Username}})
discoverdn     true
groupfilter    (&(objectClass=group)(member={{.UserDN}}))
```

Testar autenticação diretamente no AD:

```bash
ldapwhoami -x \
  -H ldap://192.168.31.24:389 \
  -D "jmsalles@jmsalles.homelab.br" \
  -W
```

### 28.2 Erro permission denied no login LDAP

Erro:

```text
Code: 403
permission denied
```

Possíveis causas:

```text
Token atual sem permissão para consultar/configurar auth method
Tentativa de login ou comando executado sem contexto correto
Método LDAP acessível, mas policy/mapeamento ainda não aplicado corretamente
```

Faça login com token administrativo:

```bash
vault login
```

Valide:

```bash
vault token lookup
```

Depois valide o mapeamento:

```bash
vault read auth/ldap/groups/GRP_HOMELAB_ADMINS
```

```bash
vault read auth/ldap/users/jmsalles
```

### 28.3 Login funciona, mas usuário fica sem permissão

Se o login LDAP funciona, mas o usuário não consegue acessar paths do Vault, o problema normalmente está na policy.

Valide o token:

```bash
vault token lookup
```

Verifique se aparece:

```text
homelab-vault-admin
```

Valide grupo mapeado:

```bash
vault read auth/ldap/groups/GRP_HOMELAB_ADMINS
```

Valide se o usuário pertence ao grupo no AD:

```bash
ldapsearch -x \
  -H ldap://192.168.31.24:389 \
  -D "svc-ldap-bind@jmsalles.homelab.br" \
  -W \
  -b "OU=Homelab,DC=jmsalles,DC=homelab,DC=br" \
  "(sAMAccountName=jmsalles)" \
  memberOf
```

### 28.4 Grupo do AD não é reconhecido

Valide o nome exato do grupo:

```bash
ldapsearch -x \
  -H ldap://192.168.31.24:389 \
  -D "svc-ldap-bind@jmsalles.homelab.br" \
  -W \
  -b "OU=Groups,OU=Homelab,DC=jmsalles,DC=homelab,DC=br" \
  "(cn=GRP_HOMELAB_ADMINS)" \
  cn distinguishedName
```

O nome usado no Vault deve bater com o `cn` do grupo:

```bash
vault write auth/ldap/groups/GRP_HOMELAB_ADMINS policies=homelab-vault-admin
```

### 28.5 Vault não conecta no AD

Teste a porta:
