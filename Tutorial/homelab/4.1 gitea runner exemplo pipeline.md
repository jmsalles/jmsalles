# Guia de automação de build, push e deploy do app-hello-homelab com Gitea Actions

## 1. Objetivo

Este documento descreve a automação do deploy da aplicação `app-hello-homelab` usando:

* Gitea
* Gitea Actions
* Gitea Runner
* Docker HML
* Nexus Registry
* Docker Compose

O objetivo é permitir que, a cada `push` no branch `main`, a pipeline execute automaticamente:

```text
Checkout do código
Build da imagem Docker
Login no Nexus Registry
Push da imagem para o registry
Criação da estrutura de deploy no Docker HML
Geração automática do docker-compose.yml
Pull da imagem publicada
Deploy do container
Validação da aplicação
Logout do registry
```

Nesta etapa **não será criada entrada DNS** e **não será criada entrada no Nginx reverse proxy**.

A aplicação ficará acessível diretamente pela porta publicada no Docker HML:

```text
http://IP_DO_DOCKER_HML:18080
```

No ambiente atual:

```text
http://192.168.31.37:18080
```

## 16. Ajustar config.yaml do Gitea Runner

Acesse o diretório do runner:

```bash
cd /home/jmsalles/gitea-runner
```

Edite o arquivo:

```bash
vim /home/jmsalles/gitea-runner/config.yaml
```

No bloco `container`, deixe conforme abaixo:

```yaml
container:
  network: ""
  privileged: false
  options: "-v /home/jmsalles/gitea-runner/certs:/certs:ro -v /home/jmsalles/docker-apps:/deploy -e GIT_SSL_CAINFO=/certs/jmsalles-homelab-rootCA.crt -e NODE_EXTRA_CA_CERTS=/certs/jmsalles-homelab-rootCA.crt"
  workdir_parent:
  valid_volumes:
    - /home/jmsalles/gitea-runner/certs
    - /home/jmsalles/docker-apps
```

Atenção:

Não adicionar manualmente o Docker socket neste bloco.

Não usar:

```text
-v /var/run/docker.sock:/var/run/docker.sock
```

Motivo:

O Gitea Runner já injeta o socket Docker no container de job. Se ele for adicionado manualmente, pode ocorrer erro:

```text
Duplicate mount point: /var/run/docker.sock
```

## 19. Criar workflow de automação

Criar ou editar o workflow:

```bash
vim /home/jmsalles/app-hello-homelab/.gitea/workflows/build-push.yml
```

Conteúdo completo:

```yaml
name: Build, Push e Deploy Docker HML

on:
  push:
    branches:
      - main

jobs:
  build-push-deploy:
    runs-on: docker-hml

    defaults:
      run:
        shell: sh

    env:
      GIT_SSL_CAINFO: /etc/ssl/certs/ca-certificates.crt
      NODE_EXTRA_CA_CERTS: /etc/ssl/certs/ca-certificates.crt
      DEPLOY_DIR: /deploy/app-hello-homelab
      CONTAINER_NAME: app-hello-homelab
      APP_PORT: 18080

    steps:
      - name: Preparar ambiente do job
        run: |
          apk add --no-cache git nodejs npm bash ca-certificates docker-cli-compose curl
          cp /certs/jmsalles-homelab-rootCA.crt /usr/local/share/ca-certificates/jmsalles-homelab-rootCA.crt
          update-ca-certificates
          git config --global http.sslCAInfo /etc/ssl/certs/ca-certificates.crt
          git config --global http.sslVerify true
          git --version
          node --version
          docker version
          docker compose version

      - name: Checkout do codigo
        uses: actions/checkout@v4

      - name: Validar workspace
        run: |
          echo "Diretorio atual:"
          pwd

          echo "Arquivos na raiz do repositorio:"
          ls -lah

          echo "Validando Dockerfile:"
          test -f Dockerfile || {
            echo "ERRO: Dockerfile nao encontrado no workspace do job"
            find . -maxdepth 3 -type f
            exit 1
          }

          cat Dockerfile

      - name: Definir variaveis da imagem
        run: |
          echo "IMAGE_NAME=${{ secrets.REGISTRY_URL }}/app-hello-homelab" >> $GITEA_ENV
          echo "IMAGE_TAG=${{ gitea.sha }}" >> $GITEA_ENV

      - name: Login no Nexus Registry
        run: |
          echo "${{ secrets.REGISTRY_PASSWORD }}" | docker login "${{ secrets.REGISTRY_URL }}" -u "${{ secrets.REGISTRY_USER }}" --password-stdin

      - name: Build da imagem
        run: |
          docker build -t "${IMAGE_NAME}:${IMAGE_TAG}" -t "${IMAGE_NAME}:latest" .

      - name: Push da imagem
        run: |
          docker push "${IMAGE_NAME}:${IMAGE_TAG}"
          docker push "${IMAGE_NAME}:latest"

      - name: Criar estrutura de deploy
        run: |
          mkdir -p "${DEPLOY_DIR}"/{logs,config}
          ls -lah /deploy
          ls -lah "${DEPLOY_DIR}"

      - name: Gerar docker-compose.yml de deploy
        run: |
          cat > "${DEPLOY_DIR}/docker-compose.yml" <<EOF
          services:
            app-hello-homelab:
              image: ${IMAGE_NAME}:latest
              container_name: ${CONTAINER_NAME}
              restart: unless-stopped
              ports:
                - "${APP_PORT}:80"
          EOF

          cat "${DEPLOY_DIR}/docker-compose.yml"

      - name: Deploy no Docker HML
        run: |
          docker compose \
            --project-name app-hello-homelab \
            -f "${DEPLOY_DIR}/docker-compose.yml" \
            pull

          docker compose \
            --project-name app-hello-homelab \
            -f "${DEPLOY_DIR}/docker-compose.yml" \
            up -d

      - name: Validar deploy
        run: |
          docker ps --filter "name=${CONTAINER_NAME}"

          docker compose \
            --project-name app-hello-homelab \
            -f "${DEPLOY_DIR}/docker-compose.yml" \
            ps

          sleep 3

          docker exec "${CONTAINER_NAME}" wget -qO- http://127.0.0.1 || {
            echo "ERRO: aplicacao nao respondeu dentro do container"
            docker logs "${CONTAINER_NAME}" --tail=50
            exit 1
          }

      - name: Logout do registry
        run: |
          docker logout "${{ secrets.REGISTRY_URL }}"
```


* Definir `GIT_SSL_CAINFO`
* Instalar CA no container do job

Trecho usado no workflow:

```bash
cp /certs/jmsalles-homelab-rootCA.crt /usr/local/share/ca-certificates/jmsalles-homelab-rootCA.crt
update-ca-certificates
git config --global http.sslCAInfo /etc/ssl/certs/ca-certificates.crt
```

### 29.4 Erro bash não encontrado

Erro:

```text
exec: "bash": executable file not found in $PATH
```

Causa:

A label `docker-hml` usa imagem baseada em Alpine:

```text
docker:27-cli
```

Solução:

Definir shell padrão como `sh`.

```yaml
defaults:
  run:
    shell: sh
```

### 29.5 Erro de Docker socket duplicado

Erro:

```text
Duplicate mount point: /var/run/docker.sock
```

Causa:

Socket Docker montado manualmente no `container.options`.

Solução:

Não colocar isto no `container.options`:

```text
-v /var/run/docker.sock:/var/run/docker.sock
```

O socket deve estar somente no `docker-compose.yml` do runner:

```yaml
volumes:
  - /var/run/docker.sock:/var/run/docker.sock
```
